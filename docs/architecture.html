Executive Summary

This Terraform module deploys Microsoft Sentinel (Azure Sentinel) with AI-powered SOC automation capabilities. It provisions a Log Analytics workspace, onboards Sentinel, configures data connectors (Azure AD, Security Center, Defender ATP), deploys scheduled analytics rules for threat detection, sets up automation rules for incident response, creates SOAR playbooks via Logic Apps with AI-powered triage endpoints, and manages threat intelligence through watchlists and TI indicators. The module enables a fully automated Security Operations Center with machine-learning-driven incident classification and response.

Component Breakdown

Log Analytics Workspace

Core data store with configurable SKU (PerGB2018, Free, Standalone, PerNode) and retention (30-730 days). Serves as the foundation for all Sentinel operations.

Data Connectors

Three toggleable connectors ingest security telemetry: Azure AD (sign-in/audit logs), Azure Security Center (security alerts), and Microsoft Defender ATP (endpoint detection).

Scheduled Analytics Rules

Custom KQL-based detection rules with configurable severity, frequency, and lookback. Default rules detect brute-force logins, impossible travel, and suspicious PowerShell execution. Rules auto-create incidents with entity-based grouping.

Automation Rules

Condition-based rules that automatically modify incident properties (e.g., escalate severity). Triggered by incident severity matching.

SOAR Playbooks (Logic Apps)

HTTP-triggered Logic App workflows that respond to Sentinel incidents. Each playbook can optionally invoke an AI endpoint for automated incident triage and classification.

Threat Intelligence

Watchlists store AI-curated threat indicators (IPs, domains) with confidence scores. TI Indicators are created for IP and domain types with STIX pattern matching.

Data Flow

  1. Security telemetry flows from Azure AD, ASC, and MDATP into the Log Analytics Workspace via data connectors
  2. Sentinel onboarding enables SIEM capabilities on the workspace
  3. Scheduled analytics rules query ingested data at configurable intervals (e.g., PT1H, PT5M)
  4. Matching queries generate incidents with severity classification
  5. MS Security Incident rules create incidents from Microsoft security product alerts
  6. Automation rules evaluate incidents and modify properties or trigger playbooks
  7. SOAR playbooks receive incident data via HTTP trigger
  8. AI triage action sends incident context to an AI/ML endpoint for classification
  9. Watchlists and TI indicators enrich investigation with known threat data

Industry Adaptation

Financial Services

Configure brute-force detection thresholds, enable all connectors, set 365-day retention for compliance, add custom KQL rules for transaction anomalies.

Healthcare

Enable HIPAA-aligned retention (180+ days), add watchlist entries for known healthcare threat actors, configure AI triage for PHI-related incidents.

Government

Set 730-day retention for FedRAMP, enable all data connectors, deploy multiple playbooks for tiered incident response workflows.

E-Commerce

Focus on impossible travel and credential stuffing rules, integrate AI triage with fraud detection endpoints, monitor for account takeover patterns.

Production Readiness Checklist

  • Configure appropriate retention period (90+ days recommended)
  • Enable all relevant data connectors for comprehensive coverage
  • Review and customize default analytics rules for your environment
  • Configure AI endpoint URL with proper authentication
  • Set up automation rules for high-severity incident escalation
  • Populate watchlists with organization-specific threat intelligence
  • Test playbook triggers with sample incidents
  • Configure RBAC: Sentinel Contributor, Logic App Contributor
  • Set up monitoring for playbook execution failures
  • Enable resource tagging for cost tracking and governance

Prerequisites
